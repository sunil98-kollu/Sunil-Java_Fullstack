In production code, always use Lombok's @RequiredArgsConstructor to avoid the boilerplate:

Field Injection (NOT Recommended ❌)
......................................
Dependencies are injected directly into variables using @Autowired.

Example
@Service
public class OrderService {

    @Autowired
    private PaymentService paymentService;

    public void placeOrder() {
        paymentService.pay();
    }
}
Problems

❌ Hard to test (cannot inject mock easily)
❌ Breaks encapsulation
❌ Hidden dependencies
❌ Cannot make fields final


2. Setter Injection (Optional Dependency)
.........................................
Dependencies are injected via setter methods.

Example
@Service
public class OrderService {

    private PaymentService paymentService;

    @Autowired
    public void setPaymentService(
            PaymentService paymentService) {

        this.paymentService = paymentService;
    }

    public void placeOrder() {
        paymentService.pay();
    }
}
Advantages
........................
✔ Good for optional dependencies
✔ Can change dependency at runtime

Disadvantages
......
❌ Object may be partially initialized
❌ Not immutable

3. Constructor Injection (BEST PRACTICE ⭐)
.........................................
Dependencies are passed through constructor.

Example
@Service
public class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

    public void placeOrder() {
        paymentService.pay();
    }
}
Advantages
..........................................
✔ Immutable (use final)
✔ Easy to test (mock injection)
✔ Fully initialized object
✔ Best for required dependencies

(Lombok generates constructor automatically)

y construtor preffered 
...........................

Constructor injection is preferred over field and setter injection because it ensures mandatory dependencies are provided at object creation time, supports immutability, and improves testability and maintainability.


@RequiredArgsConstructor // generates constructor for all final fields
public class OrderService {
private final UserService userService;
private final PaymentService paymentService;
// No @Autowired anywhere. Spring finds the Lombok-generated constructor.
// Both fields are final. This class is safe, testable, and clean.

What the interviewer expects:
"When Spring finds multiple beans of the same type, it throws
NoUniqueBeanDefinitionException at startup. @Primary marks one implementation
as the default for type-based injection. @Qualifier resolves a specific bean
by name when the default isn't what you need. For large codebases, I'd prefer
custom qualifier annotations over string-based @Qualifier because they're
type-safe, refactor-friendly, and navigable in IDEs.
